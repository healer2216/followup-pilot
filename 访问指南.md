# FollowUp Pilot - 访问指南

##  项目状态

✅ **服务器运行中**: FastAPI + StepFun LLM  
✅ **端口**: http://localhost:8080  
✅ **前端**: 独立版 HTML（无 CDN 依赖）  
✅ **后端 API**: 全部正常工作  

##  如何访问前端

### 方法1：直接在浏览器打开（推荐）

1. 打开 Chrome、Firefox 或 Edge 浏览器
2. 在地址栏输入：`http://localhost:8080`
3. 按 Enter 键

### 方法2：使用 curl 测试

```bash
# 测试首页
curl http://localhost:8080/ | head -10

# 测试健康检查
curl http://localhost:8080/api/health

# 测试解析功能
curl -X POST http://localhost:8080/api/parse \
  -H 'Content-Type: application/json' \
  -d '{"text":"患者张三，男，65岁。诊断：2型糖尿病、高血压3级。用药：二甲双胍 500mg bid"}'
```

## 📋 快速体验流程

### 1️⃣ 上传出院小结

- 页面已自动加载示例数据
- 点击 **" AI 解析"** 按钮
- 等待 2-3 秒，查看解析结果

### 2️⃣ 查看解析结果

你会看到：
- 👤 **患者信息**：姓名、性别、年龄、入出院日期
- 🏥 **诊断列表**：2型糖尿病、高血压3级等
- 💊 **用药方案**：药物名称、剂量、频次、时间
-  **生命体征**：血糖、血压等（带正常/异常标识）
- 🎯 **置信度**：AI 解析的准确率

### 3️ 生成随访计划（后续功能）

- 解析完成后，点击 **"生成随访计划"**
- 系统会调用 KnowS API 检索循证证据
- LLM 融合证据生成个性化随访时间线

### 4️ 对话随访（后续功能）

- 切换到 **"对话随访"** 页面
- 输入测试消息验证预警系统：
  - `"今天血糖 6.5"` → 🟢 绿灯（正常）
  - `"血糖有点高，8.5"` → 🟡 黄灯（关注）
  - `"血糖 16，而且头晕"` → 🔴 红灯（紧急）

## 🔧 技术栈

- **前端**: 纯 HTML + CSS + JavaScript（无框架，无 CDN）
- **后端**: FastAPI (Python)
- **LLM**: StepFun step-3.7-flash
- **循证检索**: KnowS API（6大数据源）
- **存储**: JSON 文件持久化

## 🐛 故障排查

### 问题1：浏览器显示空白

**原因**: 服务器未启动或端口被占用

**解决**:
```bash
# 检查服务器是否运行
ps aux | grep "uvicorn app.main:app" | grep -v grep

# 如果没有输出，重启服务器
cd /root/study/项目开发/followup-pilot
pkill -f "uvicorn app.main:app"
nohup /root/anaconda3/bin/python3 -m uvicorn app.main:app --host 0.0.0.0 --port 8080 &
sleep 3

# 测试
curl http://localhost:8080/api/health
```

### 问题2：API 返回错误

**原因**: LLM API Key 配置错误或 KnowS API 限流

**解决**:
```bash
# 检查 .env 文件
cat /root/study/项目开发/followup-pilot/.env

# 确认 StepFun API Key 正确
# STEPFUN_API_KEY=4iGK49lIqkUlp1unQUhIQpjaEUiBfFaerkAVux0zb7o2N2jVPmBFtaCH5aZ4hcBhU
```

### 问题3：前端样式不显示

**原因**: 浏览器缓存或 CSS 加载失败

**解决**:
- 按 `Ctrl + Shift + R` 强制刷新（Chrome）
- 按 `Ctrl + F5` 清除缓存刷新（Firefox）
- 打开开发者工具（F12），查看 Console 是否有错误
