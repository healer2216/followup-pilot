# FollowUp Pilot 功能架构说明书

## 一、项目概述

### 1.1 项目定位
FollowUp Pilot（循证随访管家）是一个 AI 驱动的出院患者随访管理系统，基于大语言模型（LLM）和医学循证检索技术，为出院患者提供智能化的随访计划生成、对话式健康监测和循证医学支持。

### 1.2 核心价值
- **智能化解析**：自动从出院小结中提取结构化信息
- **循证决策**：基于 KnowS 医学证据库的检索增强生成（RAG）
- **多 Agent 协作**：分诊官、规划师、审核员协同工作
- **实时预警**：指标趋势分析与三级预警机制
- **安全护栏**：四层安全防护确保医疗建议合规

### 1.3 技术栈
- **后端框架**：FastAPI + Uvicorn
- **LLM 供应商**：StepFun (step-3.7-flash)，支持 OpenAI/通义千问/DeepSeek 切换
- **医学循证**：KnowS Evidence Search API（6 类证据源）
- **数据验证**：Pydantic v2
- **前端**：HTML5 + Tailwind CSS + Lucide Icons（CDN）
- **会话存储**：JSON 文件存储（SessionStore）

### 2.2 核心模块说明

#### 2.2.1 LLM Gateway (`app/tools/llm_gateway.py`)
**职责**：统一封装对不同 LLM 供应商的调用
- 支持多供应商切换（OpenAI/通义千问/DeepSeek/StepFun）
- 提供 `chat_completion()` 方法
- 支持 JSON Mode 结构化输出
- 性能监控日志记录

**关键配置**：
```python
LLM_PROVIDER = "stepfun"
LLM_MODEL = "step-3.7-flash"
```

#### 2.2.2 KnowS Client (`app/tools/knows_search.py`)
**职责**：医学循证检索 API 客户端
- 支持 6 类证据源：英文论文、中文论文、会议摘要、临床指南、临床试验、药品说明书
- 串行调用策略（0.4s 间隔避免限流）
- 支持匿名调用（3 QPS 限流）

**证据源枚举**：
```python
class EvidenceSource(str, Enum):
    PAPER_EN = "paper_en"
    PAPER_CN = "paper_cn"
    MEETING = "meeting"
    GUIDE = "guide"
    TRIAL = "trial"
    PACKAGE_INSERT = "package_insert"
```

#### 2.2.3 Parser Service (`app/services/parser.py`)
**职责**：出院小结结构化解析
- 使用 LLM + JSON Mode 提取患者信息
- System Prompt 指导字段映射
- 置信度评估（基于字段完整性）
- 返回 `DischargeSummary` 对象

**解析流程**：
```
出院小结文本 → LLM 结构化提取 → JSON 解析 → Pydantic 验证 → DischargeSummary
```

#### 2.2.4 Evidence Service (`app/services/evidence.py`)
**职责**：循证检索服务
- 构建 PICO 查询（Patient/Intervention/Comparison/Outcome）
- 并行调用 KnowS 多个证据源
- 证据分级（A-E 级）
- 按需检索（对话中动态补充证据）

**核心方法**：
- `search_for_plan(summary)` - 为随访计划生成检索证据
- `search_on_demand(query)` - 对话中按需检索

#### 2.2.5 Planner Service (`app/services/planner.py`)
**职责**：随访计划生成
- 基于出院小结和循证证据生成时间线
- LLM 生成个性化任务描述
- 注入证据引用和分级标签
- 返回 `FollowUpPlan` 对象

**计划结构**：
```
FollowUpPlan
├── timeline: List[TimelineDay]
│   ├── day_number: int
│   ├── date: str
│   └── tasks: List[FollowUpTask]
│       ├── task_type: medication/monitoring/diet/activity/checkup
│       ├── title: str
│       ├── description: str
│       └── evidence_refs: List[str]
```

#### 2.2.6 Chatbot Service (`app/services/chatbot.py`)
**职责**：对话随访引擎
- 数值指标提取（LLM 轻量调用）
- 预警等级评估（AlertEngine）
- 循证证据补充检索
- LLM 生成个性化回复
- 会话上下文管理

**工作流程**：
```
用户消息 → 提取数值 → 预警评估 → 按需检索证据 → LLM 生成回复 → 返回
```

**预警等级**：
- GREEN：正常范围
- YELLOW：需要关注（略超阈值）
- RED：紧急处理（严重超标或危险症状）

**预警阈值示例**：
```python
RED_THRESHOLDS = {
    "fasting_glucose": {"high": 16.7, "unit": "mmol/L"},
    "blood_pressure_systolic": {"high": 180, "unit": "mmHg"},
}
YELLOW_THRESHOLDS = {
    "fasting_glucose": {"high": 10.0, "unit": "mmol/L"},
    "blood_pressure_systolic": {"high": 140, "unit": "mmHg"},
}
```

### 3.2 关键模型详解

#### DischargeSummary（出院小结）
```python
class DischargeSummary(BaseModel):
    patient_name: str                    # 患者姓名（必填）
    gender: str                          # 性别（必填）
    age: int                             # 年龄（必填）
    admission_date: Optional[date]       # 入院日期（可选）
    discharge_date: Optional[date]       # 出院日期（可选）
    diagnoses: List[str]                 # 诊断列表
    medications: List[Medication]        # 用药方案
    vital_signs: List[VitalSign]         # 检验指标
    chief_complaint: str                 # 主诉
    treatment_summary: str               # 治疗经过
    discharge_instructions: str          # 出院医嘱
    confidence: float                    # 解析置信度 (0.0-1.0)
```

#### FollowUpPlan（随访计划）
```python
class FollowUpPlan(BaseModel):
    patient_id: str                      # 患者ID
    generated_at: datetime               # 生成时间
    summary: DischargeSummary            # 关联的出院小结
    timeline: List[TimelineDay]          # 时间线
    evidence_stats: dict                 # 证据统计
    risk_profile: dict                   # 风险画像
    review_verdict: Optional[ReviewVerdict]  # 审核结论
    review_scores: dict                  # 审核评分
```

#### ChatSession（会话上下文）
```python
class ChatSession(BaseModel):
    session_id: str                      # 会话ID
    patient_id: str                      # 患者ID
    created_at: datetime                 # 创建时间
    discharge_summary: Optional[DischargeSummary]  # 出院小结
    plan: Optional[FollowUpPlan]         # 关联的随访计划
    messages: List[ChatMessage]          # 历史消息
    current_alert: AlertLevel            # 当前预警等级
    vital_records: List[dict]            # 指标记录 [{date, type, value}]
```

## 四、前端页面

### 4.1 页面清单

| 页面 | 文件 | 功能 |
|-----|------|------|
| 首页 | `frontend/ui-mockup.html` | 三合一单页应用（SPA） |
| 上传解析页 | 同文件 Tab 1 | 粘贴出院小结文本，AI 解析展示 |
| 随访计划页 | 同文件 Tab 2 | 时间线展示随访任务 |
| 对话随访页 | 同文件 Tab 3 | 聊天界面，实时预警 |

### 4.2 前端技术栈
- **HTML5**：语义化标签
- **Tailwind CSS**：实用优先 CSS 框架（CDN）
- **Lucide Icons**：图标库（CDN）
- **JavaScript Fetch API**：调用后端 REST API
- **LocalStorage**：会话状态持久化（待实现）

### 4.3 前端交互流程

#### 页面 1：上传解析
```
用户粘贴出院小结文本 
  → 点击"AI 解析"按钮 
  → 调用 POST /api/parse 
  → 展示解析结果（患者信息、诊断、用药、指标）
  → 显示风险画像（风险等级、风险因素、特殊注意事项）
```

#### 页面 2：随访计划
```
点击"生成随访计划"按钮 
  → 调用 POST /api/plan 
  → 展示时间线（纵向排列）
  → 每个时间节点展开显示任务卡片
  → 任务卡片显示循证引用标签（A/B/C/D/E 级）
```

#### 页面 3：对话随访
```
用户在输入框输入消息 
  → 点击发送或按回车 
  → 调用 POST /api/chat 
  → 显示 AI 回复气泡
  → 根据 alert_level 显示预警颜色（绿/黄/红）
  → 展示循证证据引用（折叠块）
  → 更新顶部预警状态徽章
```

## 六、已知限制与后续优化

### 6.1 当前限制
1. **随访计划生成**：目前返回 Mock 数据，未实现真实 RAG Pipeline 和多 Agent 编排
2. **会话存储**：使用内存字典，重启后丢失（应改为数据库或 Redis）
3. **患者画像**：风险评级为硬编码规则，未实现智能分析
4. **证据分级**：未实现自动化 A-E 分级引擎
5. **安全护栏**：缺少药物安全、诊断安全、矛盾检测等四层防护

### 6.2 性能优化建议
1. **LLM 缓存**：对相同查询启用缓存，避免重复调用
2. **流式输出**：对话随访改用 Server-Sent Events (SSE) 实现流式回复
3. **异步检索**：KnowS 检索改为真正并行（当前为串行）
4. **前端加载动画**：添加骨架屏和进度提示，改善用户体验

### 6.3 功能扩展路线
1. **多模态支持**：支持上传出院小结图片，OCR 识别后解析
2. **语音交互**：集成语音转文字（ASR）和文字转语音（TTS）
3. **移动端适配**：开发微信小程序或 React Native App
4. **医生端后台**：医生可查看患者随访数据，调整随访计划
5. **数据分析看板**：统计患者依从性、指标趋势、预警分布
