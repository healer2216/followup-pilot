# AI 出院随访管家（循证版本）— 需求与开发文档

> 黑客松竞赛项目 · 单人开发 · 48小时交付

## 1. 产品定位

### 1.1 一句话描述
患者出院后，AI 根据出院小结自动生成**循证驱动的个性化随访计划**，通过对话评估恢复情况，异常时预警。

### 1.2 解决什么问题
| 痛点 | 现状 | 我们的方案 |
|------|------|-----------|
| 出院后不知所措 | 患者拿到出院小结但不理解 | AI 用通俗语言解读 + 生成时间线 |
| 随访计划千篇一律 | 医生没时间写个性化方案 | AI 基于循证证据自动生成 |
| 漏服/忘复查 | 靠记忆或纸质提醒 | 智能提醒 + 对话式随访 |
| 异常发现晚 | 患者不知道哪些症状危险 | AI 实时评估 + 红黄绿灯预警 |

### 1.3 核心差异化：循证医学
每条建议都标注证据来源，不是 LLM"编"出来的：
- 用药提醒 → 引用 **药品说明书**（KnowS `package_insert`）
- 复查建议 → 引用 **临床指南**（KnowS `guide`）
- 症状评估 → 引用 **医学论文**（KnowS `paper_cn` / `paper_en`）

## 2. 用户故事与核心流程

### 2.1 主流程（Demo 演示路径）

```
用户上传出院小结
       ↓
  [Step 1] AI 解析出院小结
  - 提取：诊断、手术、用药、注意事项
  - 结构化展示给患者确认
       ↓
  [Step 2] 循证检索（KnowS API）
  - 查指南：该病种的随访规范
  - 查说明书：用药注意事项
  - 查论文：恢复期关键指标
       ↓
  [Step 3] 生成随访计划
  - 时间线视图：第1天/第1周/第2周/第1月...
  - 每条计划标注循证来源（"根据XX指南建议"）
       ↓
  [Step 4] 对话式随访
  - AI 扮演随访护士，主动询问恢复情况
  - 患者回答后，AI 评估是否正常
  - 异常 → 红/黄/绿灯预警
  - 严重异常 → 建议立即就医
```

### 2.2 用户故事

**US-1：上传与解析**
> 作为患者，我想拍照/上传出院小结，AI 帮我读懂里面的内容。

**US-2：查看随访计划**
> 作为患者，我想看到清晰的时间线，知道什么时候该做什么。

**US-3：对话式随访**
> 作为患者，我想随时和 AI 聊聊我的恢复情况，得到专业反馈。

**US-4：异常预警**
> 作为患者，当我的症状有危险信号时，AI 应该明确告诉我需要就医。

## 3. 功能模块设计

### 3.1 模块总览

| 模块 | 功能 | 优先级 | 预计工时 |
|------|------|--------|---------|
| M1 出院小结解析 | 上传文本/图片 → LLM 提取结构化信息 | P0 | 2h |
| M2 循证检索 | 调用 KnowS 检索指南/论文/说明书 | P0 | 2h |
| M3 随访计划生成 | 基于解析结果 + 循证证据 → 生成时间线计划 | P0 | 3h |
| M4 对话式随访 | AI 对话评估恢复情况 + 预警 | P0 | 3h |
| M5 前端界面 | 上传/计划展示/对话/预警 | P0 | 4h |
| M6 提醒系统 | 定时提醒服药/复查（Demo 模拟） | P1 | 1h |

### 3.2 M1：出院小结解析

**输入**：患者粘贴的出院小结文本（或 OCR 后的文本）

**LLM 提取结构**：
```json
{
  "patient_name": "张三",
  "diagnosis": "2型糖尿病、高血压3级",
  "hospitalization_dates": {"admission": "2026-06-20", "discharge": "2026-06-28"},
  "procedures": ["冠状动脉造影术"],
  "medications": [
    {"name": "二甲双胍", "dosage": "500mg", "frequency": "每日两次", "purpose": "控制血糖"},
    {"name": "氨氯地平", "dosage": "5mg", "frequency": "每日一次", "purpose": "降压"}
  ],
  "precautions": ["低盐低脂饮食", "定期监测血糖血压"],
  "followup_recommendation": "出院后2周门诊复查",
  "lab_results": {"fasting_glucose": "7.8 mmol/L", "blood_pressure": "145/92 mmHg"}
}
```

**实现方案**：
- 使用 LLM（通义千问/OpenAI）+ 结构化输出（JSON Mode）
- Prompt 设计：给出出院小结样例 → 要求提取上述字段
- 人工确认环节：解析结果展示给患者，支持修改

### 3.3 M2：循证检索（KnowS 集成）

基于 M1 的解析结果，自动构造查询：

| 查询目的 | KnowS 源 | 示例 Query |
|---------|----------|-----------|
| 病种随访规范 | `guide` | "2型糖尿病出院后随访管理指南" |
| 用药注意事项 | `package_insert` | "二甲双胍 注意事项 不良反应" |
| 恢复期指标 | `paper_cn` | "2型糖尿病 出院后 血糖监测 随访" |
| 最新研究 | `paper_en` | "type 2 diabetes post-discharge follow-up management" |

**调用规则**：
- 串行调用（避免 429 限流），间隔 0.4s
- 匿名 3 QPS，有 API Key 10 QPS
- API 地址：`https://api.nullht.com/v1`
- 每个源最多返回 5-40 条，取 Top 3-5 最相关的

### 3.5 M4：对话式随访

**AI 角色设定**：专业随访护士，温暖但专业

**对话流程**：
```
AI: 您好！我是您的随访助手。今天想了解一下您的恢复情况。
    请问今天空腹血糖测了吗？数值是多少？

患者: 测了，6.8

AI: ✅ 空腹血糖 6.8 mmol/L，在目标范围内（4.4-7.0），控制得不错！
    继续保持目前的用药和饮食方案。
    [引用：中国2型糖尿病防治指南 - 空腹血糖目标 4.4-7.0 mmol/L]

    另外，最近有没有感觉心慌、出汗、手抖的情况？

患者: 昨天下午有点心慌

AI: ⚠️ 下午心慌可能与低血糖有关。请问当时距上次进食多久？
    ...
    建议在下午3-4点加餐一次（如一小杯酸奶或几块饼干）。
    如果频繁出现，请提前到门诊复查。
    [引用：二甲双胍说明书 - 常见不良反应包括低血糖]
```

**预警等级**：
| 等级 | 颜色 | 触发条件 | AI 行为 |
|------|------|---------|--------|
| 正常 | 🟢 绿 | 指标在目标范围 | 肯定 + 继续观察 |
| 关注 | 🟡 黄 | 指标轻度异常 | 建议调整 + 持续监测 |
| 警告 | 🔴 红 | 指标严重异常/危险症状 | 强烈建议立即就医 |

## 5. API 接口设计

### 5.1 POST /api/parse — 解析出院小结

```json
// Request
{ "text": "出院小结全文..." }

// Response
{
  "structured": { "diagnosis": "...", "medications": [...], ... },
  "raw_text": "...",
  "confidence": 0.92
}
```

### 5.2 POST /api/plan — 生成随访计划

```json
// Request
{ "structured": { /* parse 返回的结构化数据 */ } }

// Response
{
  "plan_id": "plan_001",
  "timeline": [
    {
      "day": 1,
      "label": "出院第1天",
      "tasks": [
        {
          "type": "medication",
          "title": "...",
          "detail": "...",
          "evidence": { "source": "package_insert", "title": "...", "snippet": "..." }
        }
      ]
    }
  ],
  "evidence_summary": "共检索到 X 条循证证据，引用 Y 条"
}
```

### 5.3 POST /api/chat — 对话随访

```json
// Request
{
  "plan_id": "plan_001",
  "message": "我今天空腹血糖 6.8",
  "history": [/* 历史消息 */]
}

// Response
{
  "reply": "✅ 空腹血糖 6.8 mmol/L，在目标范围内...",
  "alert_level": "green",        // green | yellow | red
  "alert_detail": null,          // red/yellow 时填写
  "evidence_refs": [
    { "source": "guide", "title": "...", "snippet": "..." }
  ],
  "suggest_actions": ["继续保持当前方案", "建议记录饮食日志"]
}
```

### 5.4 GET /api/health — 健康检查

```json
{ "status": "ok", "knows_api": true, "llm_api": true }
```

## 8. 技术亮点（评委关注点）

1. **循证驱动**：每条建议都引用真实医学文献/指南/说明书
2. **KnowS 集成**：使用 6 大医学数据源（论文、指南、药品说明书等）
3. **安全预警**：红黄绿灯三级预警，保障患者安全
4. **免责声明**：全链路标注"仅供循证参考，不替代临床医嘱"
5. **可解释性**：AI 说的每句话都能追溯到证据来源

## 9. 风险与应对

| 风险 | 概率 | 应对 |
|------|------|------|
| KnowS API 限流 429 | 中 | 串行调用 + 0.4s 间隔 + 重试机制 |
| LLM 输出不稳定 | 中 | 结构化 Prompt + 输出校验 + 重试 |
| 出院小结格式多样 | 高 | 先支持纯文本粘贴，OCR 作为 P2 |
| 对话中患者描述模糊 | 高 | AI 追问机制 + 预设常见问题 |
| 演示时网络问题 | 低 | 准备录屏 + 本地 mock 数据 |
