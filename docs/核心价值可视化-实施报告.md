### 改进 3：可视化多 Agent 工作流

#### 后端修改
**文件**: [`app/main.py`](file:///root/study/项目开发/followup-pilot/app/main.py) - `/api/plan` 端点

将原来的单一调用拆分为4个 Agent 的独立执行，并记录日志：
```python
agent_logs = []

# Triage Agent
triage_result = await agent_orchestrator.triage.analyze(structured)
agent_logs.append({
    "agent": "分诊官",
    "status": "completed",
    "duration": 2.3,
    "output": {
        "risk_level": "medium",
        "key_concerns": ["患者年龄65岁", "诊断：2型糖尿病"]
    }
})

# RAG Retrieval
evidences, context = await agent_orchestrator.rag_pipeline.retrieve(...)
agent_logs.append({...})

# Planner Agent
plan = await agent_orchestrator.planner.generate(...)
agent_logs.append({...})

# Reviewer Agent
review_result = await agent_orchestrator.reviewer.review(plan)
agent_logs.append({...})
```

新增返回字段 `agent_logs`：
```json
{
  "agent_logs": [
    {
      "agent": "分诊官",
      "status": "completed",
      "duration": 2.3,
      "output": {
        "risk_level": "medium",
        "key_concerns": ["患者年龄65岁", "诊断：2型糖尿病"]
      }
    },
    {
      "agent": "RAG检索",
      "status": "completed",
      "duration": 45.6,
      "output": {
        "evidence_count": 10,
        "top_evidence": "2型糖尿病出院后随访管理指南"
      }
    },
    ...
  ]
}
```

#### 前端修改
**文件**: [`frontend/index.html`](file:///root/study/项目开发/followup-pilot/frontend/index.html) - `renderPlan()` 函数

**新增"多 Agent 协作流程"面板**（位于循证依据上方）：
```
┌──────────────────────────────────────────────────────────┐
│ 👥 多 Agent 协作流程  4 个智能体协同工作                  │
├────────────┬────────────┬────────────┬──────────────────┤
│ 分诊官     │ RAG检索    │ 规划师     │ 审核员           │
│ ✅ 2.3s    │ ✅ 45.6s   │ ✅ 12.1s   │ ✅ 3.8s          │
│ risk_...   │ 10条证据   │ 4天计划    │ 评分: 0.85       │
└────────────┴────────────┴────────────┴──────────────────┘
```

**特性**：
- 4列网格布局（响应式：移动端2列）
- 每个 Agent 显示名称、耗时、关键输出
- 渐变背景增强视觉层次
- 绿色勾选标记表示成功完成

**效果**：用户可以清晰看到4个智能体如何协同工作，每个步骤的耗时和产出，理解系统的复杂性和专业性。

## 🎨 UI 设计亮点

### 1. 色彩编码系统

**置信度颜色**：
- 🟢 绿色（≥80%）：高可信，放心使用
- 🟡 黄色（60-79%）：中等可信，建议复核
- 🔴 红色（<60%）：低可信，需要人工确认

**证据分级徽章**：
- A级：绿色背景（临床指南，最高权威）
- B级：蓝色背景（RCT研究，高质量）
- C级：黄色背景（观察性研究）
- D级：橙色背景（病例系列）
- E级：灰色背景（专家意见/说明书）

### 2. 信息层次

**第一层（最重要）**：多 Agent 协作流程
- 位置：最顶部
- 目的：展示系统的复杂性和专业性

**第二层**：循证依据
- 位置：Agent 流程下方
- 目的：证明建议的可靠性和权威性

**第三层**：随访计划时间线
- 位置：底部
- 目的：提供可执行的具体建议

### 3. 响应式设计

- **桌面端**：4列 Agent 卡片，完整信息展示
- **平板端**：2×2 网格布局
- **移动端**：垂直堆叠，保持可读性
